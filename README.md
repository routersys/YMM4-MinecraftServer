# MC サーバーホスト for YMM4

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](#)
[![.NET](https://img.shields.io/badge/.NET-10.0-purple.svg)](#)
[![Release](https://img.shields.io/github/v/release/routersys/YMM4-MinecraftServer.svg)](https://github.com/routersys/YMM4-MinecraftServer/releases)

YukkuriMovieMaker4 (YMM4) 上で動作する **Minecraft サーバー管理ツールプラグイン**です。
Vanilla・Paper・Fabric・Forge・Bedrock の5種類のサーバータイプに対応し、サーバーJARのダウンロードから起動・停止・コンソール操作・プラグイン/MOD管理・ファイル編集までを単一のUIで行います。

![Image](https://github.com/routersys/YMM4-MCServer/blob/main/docs/MCHost.png)

---

## 概要

本プラグインは、YMM4のツールプラグイン (`IToolPlugin`) として実装されたMinecraftサーバー管理インターフェースです。サーバープロセスの起動・停止・コンソール操作を行うほか、各ディストリビューション公式APIからサーバーJARを取得してSHA-1またはSHA-256でその整合性を検証します。ロールベースの認可システム、監査ログ、構造化ログ、スケジューラーを備えます。

---

## 対応サーバータイプ

| サーバータイプ | 配布元API | 整合性検証方式 |
|---|---|---|
| Vanilla | piston-meta.mojang.com | SHA-1 (Mojangメタデータ) |
| Paper | fill.papermc.io (v3) / api.papermc.io (v2) | SHA-256 |
| Fabric | meta.fabricmc.net | リモートチェックサム / コンテンツハッシュ |
| Forge | maven.minecraftforge.net | リモートチェックサム / コンテンツハッシュ |
| Bedrock | BDS-Versions (GitHub) | リモートチェックサム / コンテンツハッシュ |

## 主な機能

### 1. サーバーJAR管理

- 各ディストリビューション公式APIからサーバーJARを自動取得します。
- JARダウンロード前後にハッシュ検証を実施し、整合性を確認します。
- 同一バージョン・ビルドが既にインストールされている場合は再ダウンロードをスキップします。
- 安全更新バックアップ機能を有効にすると、JARの更新前にサーバーディレクトリ全体をZIPアーカイブとして保存します。ダウンロードまたは検証が失敗した場合はバックアップから自動復元します。

### 2. Javaランタイム解決

- MinecraftバージョンとMojangメタデータを照合し、必要なJavaメジャーバージョンを決定します。
- ユーザー設定のJavaパスが存在する場合はそれを使用します。存在しない場合はAdoptium (Eclipse Temurin) からJREを自動ダウンロードして展開します。
- メタデータ取得に失敗した場合は、Minecraftバージョン番号に基づくフォールバックテーブル (Java 8/16/17/21) を参照します。

### 3. サーバープロセス管理

- サーバープロセスをWindows Job Objectに関連付け、ホストプロセス終了時に子プロセスも終了することを保証します。
- 標準出力・標準エラー出力をリアルタイムに取得し、ログレベル (FATAL/ERROR/WARN/DEBUG) および入退室イベントに応じて色分けしてコンソールに表示します。
- ポート開放機能を有効にすると、サーバー起動時にWindowsファイアウォールルールを追加し、Open.NAT経由でUPnPポートマッピングを試行します。プロセス終了時にはこれらを削除します。

### 4. 自動再起動

- 設定で有効にすると、サーバープロセスが予期せず終了した場合に指定の遅延後に再起動します。
- 最大試行回数を超えた場合、またはEULA同意待ち状態が検出された場合は再起動を停止します。
- ユーザー操作による停止 (`StopServer`) では自動再起動は実行されません。

### 5. バージョン・ビルド選択

- 各ディストリビューション公式APIから利用可能なバージョン一覧を取得し、ComboBoxに表示します。バージョンは80件単位の遅延ロードで表示されます。
- Paperサーバーについては、ビルド番号の一覧を取得し、個別に選択できます。

### 6. プラグイン管理 (Paper専用)

- `plugins/` ディレクトリに対してJARファイルのコピー (インストール) および削除 (アンインストール) を行います。
- ダイアログからの選択とウィンドウへのドラッグ＆ドロップに対応します。

### 7. MOD管理 (Fabric / Forge)

- `mods/` ディレクトリに対してJARファイルのコピーおよび削除を行います。
- ダイアログからの選択とドラッグ＆ドロップに対応します。

### 8. サーバーファイルエディター

- `server.properties`、`eula.txt`、`whitelist.json`、`ops.json`、`banned-players.json`、`banned-ips.json` などの設定ファイルをエディター内で直接編集します。
- `.properties`、`.json`、`.yml` / `.yaml` に対してシンタックスハイライトを適用します。
- `whitelist.json`、`ops.json`、`banned-players.json` については、Mojang API経由でプレイヤーUUIDを解決し、エントリー追加・削除を行う専用UIを提供します。Bedrock版では `allowlist.json` に対応します。
- ワールドマネージャー機能では、`%AppData%\.minecraft\saves` または任意のディレクトリにあるワールドデータをサーバーに複製します。ネザー・エンドのディメンションディレクトリ (`DIM-1`、`DIM1`) を自動で識別します。

### 9. 参加者表示

- サーバーコンソール出力を正規表現で解析し、プレイヤーの参加・退室をリアルタイムに反映します。
- Mojang APIでUUIDを解決し、Crafatar または Minotar からアバター画像を取得・キャッシュします。

### 10. ロールベース認可システム

操作はロール (Viewer / Operator / Administrator) によって制限されます。

| 操作 | 最低必要ロール |
|---|---|
| サーバー作成 | Operator |
| サーバー削除 | Administrator |
| サーバー起動・停止 | Operator |
| プラグイン・MOD管理 | Operator |
| ファイル編集 | Operator |
| コンソールコマンド送信 | Operator |
| 監査ログ閲覧 | Operator |
| ポリシー管理 | Administrator |

ロールを現在のロールより上位に昇格させる場合、認証が必要です。Windows Hello (UserConsentVerifier) を優先使用し、利用できない場合はPIN (PBKDF2-SHA256、10万2000イテレーション) によるフォールバック認証を行います。

### 11. ポリシー管理

- **メンテナンスモード**: 有効にすると、サーバー停止を除くすべての操作が拒否されます。
- **サーバーロック**: 特定サーバーIDをロック対象に指定すると、そのサーバーへの操作が制限されます。ポリシー状態は `policy/policy-state.json` にJSON形式で永続化されます。

### 12. 監査ログ

- サーバー設定変更・イベント (起動、停止、プロセス終了など) を `audit/audit.log` にJSON Lines形式で記録します。

### 13. 構造化ログ

- Trace/Debug/Information/Warning/Error/Critical の6レベルに対応したJSON Linesログをファイルに出力します。
- ファイルサイズ上限・保持ファイル数・保存ディレクトリはユーザーが設定できます。
- ログビューアーでは、レベル・カテゴリ・操作種別・サーバーID・フリーワードによるフィルタリングが可能です。

### 14. ジョブオーケストレーター

- サーバーJAR取得・Java解決・ファイル書き込みなどの非同期操作をジョブとして管理します。
- ネットワークエラー・タイムアウト・ソケットエラーに対して指数バックオフ付きの再試行 (最大3回、デフォルト) を行います。
- ジョブ履歴は最大2,000件をメモリ上に保持し、運用ダッシュボードから参照できます。

### 15. スケジューラー

- Interval (一定間隔) / Daily (毎日指定時刻) / Weekly (毎週指定曜日・時刻) / SpecificDate (指定日時) の4モードでタスクを実行します。
- 実行対象はサーバーディレクトリをカレントとして起動される `.bat`、`.cmd`、`.ps1` スクリプトです。
- 実行時に `TARGET_SERVER_ID`、`TARGET_SERVER_NAME`、`TARGET_SERVER_DIR` 環境変数を設定します。

### 16. 多言語対応

日本語・英語・簡体字中国語・繁体字中国語・韓国語・スペイン語・アラビア語・インドネシア語の8言語に対応します。

## 動作要件

- **OS**: Windows 10 / 11 (64bit)
- **YukkuriMovieMaker4**: 最新版を推奨
- **ランタイム**: .NET 10.0

## インストール方法

1. リリースページから最新のプラグインファイル (`.ymme`) をダウンロードしてください。
2. YMM4が起動していないことを確認し、ダウンロードしたファイルを実行 (ダブルクリック) してインストールします。
3. YMM4を起動し、ツールメニューから **「Minecraft Server Host」** を選択します。

## 使い方ガイド

### 基本的な使用フロー

1. **サーバーを作成する**: 「新規作成」ボタンを押し、サーバータイプ・バージョン・メモリ・ポートを設定します。
2. **起動する**: 「起動」ボタンを押します。Javaが未設定の場合は自動取得、サーバーJARが未取得の場合は自動ダウンロードと整合性検証が実行されます。
3. **コンソールを操作する**: テキストボックスにコマンドを入力してEnterキーを押すか送信ボタンを押します。
4. **停止する**: 「停止」ボタンを押します。`stop` コマンドを送信後、10秒以内に終了しない場合はプロセスを強制終了します。

### Forgeのインストールについて

Forgeはインストーラー (`forge-{version}-installer.jar`) を実行してライブラリをセットアップする必要があります。初回起動時に自動でインストーラーを実行します。Java実行ファイルパスを設定済みの場合、そのJavaが使用されます。

### Bedrockサーバーについて

BDS (Bedrock Dedicated Server) はEULAへの同意が必要です。初回起動時にEULA確認ダイアログが表示されます。同意した後は再度表示されません。Bedrockサーバーはポート番号 19132 (デフォルト) を使用します。

## 注意事項

- **インターネット通信**: 本プラグインは、サーバーJARのダウンロード・バージョン一覧の取得・Javaランタイムのダウンロードのためにインターネットへアクセスします。アクセス先は各サーバーディストリビューションの公式配布元および Adoptium に限定されます。
- **管理者権限**: ポート開放機能はWindowsファイアウォールを操作するため、管理者権限が必要な場合があります。
- **バックアップ**: JARの更新時にサーバーディレクトリ全体をバックアップする設定が可能ですが、ワールドデータの保護には別途バックアップ手段を講じることを推奨します。
